1 - Bypass FTP connection information on Plugin installation or Wordpress Upgrade by adding following line in wp-config.php

define( 'FS_METHOD', 'direct');

 

2 - To troubleshoot issues we can enable Wordpress Debugging by adding following lines in WP-CONFIG.PHP

define( 'WP_DEBUG', true );
define( 'WP_DEBUG_DISPLAY', false );
//define( 'WP_DEBUG_DISPLAY', true );
define( 'WP_DEBUG_LOG', true );

The above line of code will generate log file in wp-content directory of your Webroot.

 

3 - Site and Wordpress URL can be changed by adding following lines in wp-config.php:

define('WP_SITEURL', 'http://www.domain.com');
define('WP_HOME', 'http://www.domain.com');

This is useful if you are migrating your site to new URL because these lines override the value define in wp_options table. Another method is to use SERVER variable which will set these values dynamically:

define('WP_HOME', 'http://' . $_SERVER['HTTP_HOST'] );
define('WP_SITEURL', 'http://' . $_SERVER['HTTP_HOST'] );

 

4 - PHP memory can be increased for your Wordpress application by adding following lines:

define( 'WP_MEMORY_LIMIT', '64M' );

5 - By default if we delete Post, Pages, Comments etc it is sent to Trash box where Wordpress delete them after 30 days. To change this duration you can add following line and can modify value of days as per your need. In our case we have decreased it to 15 days.

define( 'EMPTY_TRASH_DAYS', 15 );

We can disable Trash box completely by providing 0 (zero) as value in above example.

 

6 - From Wordpress 3.7 Automatic Updates are introduced.By default minor core releases and translation files are updated. Disable this feature by introducing following lines in wp-config.php:

 define( 'AUTOMATIC_UPDATER_DISABLED', true );

7 - When creating or editing Post, Wordpress automatically save revisions of your post. The default value of Auto-Save interval is 60 Seconds which can be increased or decreased by adding following line:

define( 'AUTOSAVE_INTERVAL', 180 );
